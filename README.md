# HealthyByte Intelligence

**HealthyByte Intelligence** is a cybersecurity dashboard that aggregates and categorizes the latest news, tools, vulnerabilities, and events from 13 trusted RSS feeds and the CIRCL CVE API. Designed for cybersecurity enthusiasts, professionals, and organizations, it provides a curated, deduplicated feed in JSON format, hosted on GitHub Pages and embedded in a Squarespace site for an accessible, user-friendly interface.

This repository contains the Python script to generate the dashboard data, the output JSON, a caching mechanism to prevent duplicates, and a GitHub Actions workflow for automated weekly updates, ensuring fresh content with minimal manual effort.

## Features

- **Diverse Sources**: Aggregates content from 13 RSS feeds (e.g., The Hacker News, Krebs on Security, TechCrunch, Wired) and the CIRCL CVE API for comprehensive cybersecurity coverage.
- **Smart Categorization**: Organizes items into four distinct categories:
  - **Cybersecurity Business**: Acquisitions, funding, partnerships, regulations.
  - **Security Tools**: Penetration testing tools, scanners, open-source projects.
  - **Threats and Vulnerabilities**: CVEs, malware, phishing, data breaches.
  - **Cyber Events**: Conferences, hackathons, webinars, training sessions.
- **Deduplication**: Removes duplicate stories across sources using a 90% title similarity threshold.
- **Balanced Feeds**: Limits each source to 3 items to ensure diversity, preventing any single feed (e.g., TechCrunch) from dominating.
- **Weekly Freshness**: Filters content from the last 7 days, updated weekly via GitHub Actions.
- **JSON Output**: Generates `cybersecurity_data.json` with up to 7 items per category, hosted on GitHub Pages.
- **Robust Timezone Handling**: Supports varied date formats (e.g., EST, UTC) with consistent UTC output.
- **Error Resilience**: Gracefully handles feed failures, malformed XML, and parsing issues.

## Project Structure

The repository contains the following files:

- **`sec_dashboard.py`**:
  - Python script to fetch, deduplicate, categorize, and generate dashboard data.
  - Dependencies: `feedparser`, `requests`, `python-dateutil`, `pytz`.
  - Outputs: `cybersecurity_data.json`, `cache.json`.

- **`cybersecurity_data.json`**:
  - JSON file containing categorized cybersecurity items, generated by the script.
  - Hosted at: `https://<your-username>.github.io/hb-intelligence/cybersecurity_data.json`.
  - Example format:
    ```json
    {
      "Cybersecurity Business": [
        {
          "source": "SecurityWeek",
          "title": "Cyber Firm Raises $100M",
          "link": "https://www.securityweek.com/...",
          "summary": "A cybersecurity startup secured...",
          "published": "2025-04-14T12:00:00+00:00"
        },
        ...
      ],
      "Security Tools": [...],
      "Threats and Vulnerabilities": [...],
      "Cyber Events": [...]
    }

- **`cache.json`**:
Tracks processed item IDs (URLs or CVE IDs) and the last run timestamp to avoid duplicates across runs. Generated and updated by `sec_dashboard.py`.

**Example:**
```json
{
  "items": {
    "https://thehackernews.com/2025/04/new-malware.html": "2025-04-14T10:00:00+00:00",
    "https://www.securityweek.com/cyber-firm-funding": "2025-04-13T15:30:00+00:00",
    "CVE-2025-12345": "2025-04-12T08:00:00+00:00"
  },
  "last_run": "2025-04-14T12:00:00+00:00"
}
```

**Purpose:** Ensures only new items are processed, maintaining efficiency and consistency.

## Setup

### Clone the Repository
```bash
git clone https://github.com/<your-username>/hb-intelligence.git
cd hb-intelligence
```

### Install Dependencies
```bash
pip install feedparser requests python-dateutil pytz
```

### Run the Script
```bash
python sec_dashboard.py
```
- Generates `cybersecurity_data.json` and `cache.json` in the root.
- Updates `cache.json` with new item IDs and timestamp.

### Push Files to GitHub
```bash
git add cybersecurity_data.json cache.json sec_dashboard.py .github/workflows/update-json.yml
git commit -m "Add dashboard files"
git push origin main
```

### Enable GitHub Pages
1. Go to **Settings > Pages** in your repository.
2. Set **Source** to `main`, folder to `/ (root)`.
3. Save to host `cybersecurity_data.json` at:
   `https://<your-username>.github.io/hb-intelligence/cybersecurity_data.json`

## Automation

### Weekly Updates via GitHub Actions
- **Workflow File:** `.github/workflows/update-json.yml`
- **Schedule:** Every Monday at 9 AM UTC

### Actions Performed
- Installs Python and dependencies
- Runs `sec_dashboard.py`
- Commits updated `cybersecurity_data.json` and `cache.json`

### Manual Trigger
- Go to **Actions > Update Cybersecurity JSON > Run workflow**

**To verify:**
- Check **Actions tab** for logs
- Confirm `cybersecurity_data.json` updates at:
  `https://<your-username>.github.io/hb-intelligence/cybersecurity_data.json`
- Ensure `cache.json` reflects new items and `last_run`

## Customization

### Add Feeds
Edit `RSS_FEEDS` in `sec_dashboard.py`:
```python
RSS_FEEDS["New Feed"] = "https://newfeed.com/rss"
```

### Adjust Categories
Update `CATEGORIES` dictionary and output in `update_dashboard`.

### Change Limits
Modify:
```python
MAX_ITEMS_PER_FEED = 3
MAX_ITEMS_PER_CATEGORY = 7
```

### Tweak Deduplication
Adjust threshold in `is_duplicate()` (default: `0.9`).

### After changes
```bash
python sec_dashboard.py
git add .
git commit -m "Update dashboard configuration"
git push origin main
```

## Troubleshooting

### Script Errors
- Verify dependencies: `pip install feedparser requests python-dateutil pytz`
- Check Python version: `python --version` (must be 3.8+)
- Run the script: `python sec_dashboard.py`

### Empty JSON
- Clear cache: `rm cache.json`
- Test feeds manually: `curl <feed-url>`
- Look for `Failed to fetch` errors

### Miscategorized Items
- Check `cybersecurity_data.json`
- Refine `CATEGORIES` keywords in `sec_dashboard.py`

### Actions Failure
- View logs under GitHub **Actions**
- Ensure `.github/workflows/update-json.yml` exists
- Trigger manually: **Run workflow**

### Squarespace Issues
- Verify JSON URL: `https://<your-username>.github.io/hb-intelligence/cybersecurity_data.json`
- Check browser console for fetch errors
- Ensure JS categories match feed categories
- Open an issue if problems persist
